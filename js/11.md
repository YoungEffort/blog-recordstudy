# 可维护性

### 代码约定
1. 可读性
   * 代码缩进    
   * 代码注释
        * 函数和方法
        * 大段代码
        * 复杂的算法
        *  Hack

2. 变量和函数命名
    * 变量名应为名词如 car 或 person
    * 函数名应该以动词开始，如 getName() 。返回布尔类型值的函数一般以 is 开头，如isEnable() 
    * 变量和函数都应使用合乎逻辑的名字，不要担心长度。长度问题可以通过后处理和压缩（本章后面会讲到）来缓解
  
3. 变量类型透明
    * 通过初始化指定变量类型
    * 指定数据类型的匈牙利标记法
    * 指定类型的类型注释
  
### 松散耦合
1. 解耦 HTML/JavaScript
2. 解耦 CSS/JavaScript
3. 解耦应用逻辑／事件处理程序
   
### 编程实践
1. 尊重对象所有权
2. 避免全局量
3. 避免与 null 进行比较
4. 使用常量
   * 重复值
   * 用户界面字符串
   * URLs
   * 任意可能会更改的值
